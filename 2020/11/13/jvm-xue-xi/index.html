###【Java的类加载机制】

类加载过程

加载 验证 准备 解析 初始化 使用 卸载

其中，验证 准备 解析 统称 连接

类的加载机制

  • 全盘负责：当一个类加载器负责加载某个Class时，该Class所依赖的和引用的其他Class也将由该类加载器负责载入，除非显示使用其他的类加载器来载入
  • 父类委托：先让父类加载器试图加载该类，只有在父类加载器无法加载该类时才尝试从自己的类路径加载
  • 缓存机制：缓存机制保证所有加载过的Class都会被缓存，当程序中需要使用到某个Class时，类加载器会先从缓存中寻找改Class，缓存中不存在，JVM才会读取该类对应的二进制数据，并将其转换成Class对象，存入缓存。这也是为什么修改了Class后，需要重启JVM看，程序的修改才生效

类加载方式

  1. JVM初始化加载
  2. Class.forName()
  3. ClassLoader.loadClass()

双亲委派机制

当AppClassLoader去加载一个Class时，把加载请求委派给ExtClassLoader 去加载，ExtClassLoader又委派给BootStrapLoader去加载，找不到则一次往下加载，如果AppClassLoader也找不到，则会抛出ClassNotFoundException

意义：

  • 防止出现多份同样的字节码
  • 保证Java安全稳定运行

Java的内存区域

1.程序计数器

  1. 对于多线程而言，程序计数器就是为了记录该线程让出CPU时的执行地址，当该线程再次获取到时间片的时候，就可以继续往下执行
  2. 字节码解释器通过改变程序计数器来依次来读取指令，从而实现代码的流程控制，如：顺序执行、选择、循环、异常处理
  3. 线程私有，唯一不会OutOfMemoryError 的内存区域

2.栈

  1. 和程序计数器一样，也是线程私有，生命周期和线程相同，描述的是 Java 方法执行的内存模型
  2. 为虚拟机运行 Java 方法服务
  3. 由一个个栈帧组成，每个栈帧中都拥有：局部变量表、操作数栈、动态链接、方法出口信息

3. 本地方法栈

  1. 为虚拟机使用到的 Native 方法服务
  2. 本地方法被执行时，也会在本地方法栈创建一个栈帧，存放该本地方法的局部变量表、操作数、动态链接、出口信息
  3. 线程私有

4. 堆

  1. Java 虚拟机管理的内存中最大的一块，存放对象的实例，几乎所有的对象实例以及数组都是在这里分配的
  2. 垃圾回收的主要区域；从垃圾回收的角度，由于现在的垃圾回收器都采用分代回收算法，所有 Java 堆还可以为：新生代、老年代，再细分：Eden 空间、From Survivor、To Survivor，进一步 划分是为了：更好的回收或分配内存
  3. 线程共享

5. 方法区

  1. 线程共享
  2. 存储已被 JVM 加载的类信息、变量、静态变量、即时编译器编译后的代码等
  3. 垃圾回收很少在这个区域出现，因为“性价比不高”
